🔄 Lifecycle документа

⚪ Draft
→
📄 Generated
→
✉ Sent for Sign
→
✅ Signed
→
🗃 Archived
Draft

Создана запись: сотрудник + шаблон. PDF ещё не сгенерирован.
decoration-danger в list view — требует внимания.

Generated

PDF создан и сохранён в ir.attachment. Можно скачать.
Красная подсветка — нужно подписать. Bulk generate доступен.

Sent for Sign

Отправлен через Odoo Sign. Ждём подпись обеих сторон.

Signed

Обе стороны подписали. Cron проверяет каждый час.

Archived

Документ в архиве. Доступен для чтения.
Фильтр Active скрывает архивные по умолчанию.

📜 Формат контента секций (JSON)

Поле content_en / content_local — JSON массив:

typeОписаниеПример
paragraph Обычный абзац текста {"type": "paragraph", "text": "The Parties agree..."}
bullet Пункт с номером (1.1, (a), итд.) {"type": "bullet", "label": "3.1", "text": "The Receiving Party shall..."}
callout Выделенный блок (tinted bg + левая полоса) {"type": "callout", "text": "IMPORTANT: ..."}
service_callout Блок с описанием услуги (из employee) {"type": "service_callout"}
compensation Авто-расчёт rate × (1 + tax) {"type": "compensation"}
notice_boxes Адреса для уведомлений (2 стороны) {"type": "notice_boxes"}

📖 Руководство пользователя

1 Создание шаблона
  • WS Contracts → Configuration → Contract Templates
  • Заполнить: компания, тип документа, язык
  • Настроить цвета (color picker), подписанта, адрес
  • Добавить секции во вкладке Sections
2 Заполнение данных сотрудника
  • Employees → выбрать сотрудника → вкладка "Contract Details"
  • 4 группы: Personal Passport Bank Agreement
  • Обязательные для NDA: ФИО, дата рождения, паспорт
  • Обязательные для Contract: + IBAN, rate, service description
3 Генерация документа
  • На карточке сотрудника: кнопка "Generate Documents"
  • Или: WS Contracts → Documents → New
  • Выбрать сотрудника + шаблон → Generate PDF
  • Bulk: выбрать несколько draft документов → Action → Generate PDF
4 Подпись
  • На документе: "Send for Signature" (требует Odoo Sign)
  • Создаёт sign.template + sign.request
  • Cron проверяет статус каждый час
  • Или: "Mark as Signed" для ручной подписи
5 Архивация
  • "Archive" на документе или bulk Archive
  • Фильтр "Active" по умолчанию скрывает архивные
📋 Kanban Board
  • Документы группируются по статусу
  • Progressbar показывает распределение
  • Красный = Generated (нужно подписать)
👤 Smart Buttons на сотруднике
  • "Documents" — все документы сотрудника
  • "Pending Sign" — ожидают подписи (появляется только если count > 0)
